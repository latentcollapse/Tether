# Tether v1.3 (Mar 2, 2026)

- **New:** `ttl_seconds` on `tether_send` — messages self-destruct after N seconds. Expired handles raise `E_HANDLE_EXPIRED` on resolve and are silently excluded from `tether_inbox` and `snapshot`.
- **New:** P.O. Box ownership — `tether_send` now stamps each message with `owner=to`. Pass `for_agent` to `tether_receive` to enforce it: access is denied if the handle belongs to a different agent (`E_ACCESS_DENIED`). Omitting `for_agent` remains backwards compatible.
- **New:** `E_HANDLE_EXPIRED` and `E_ACCESS_DENIED` exception types in `tether.exceptions`.
- **Migration:** Non-breaking. Existing databases gain `expires_at` and `owner` columns automatically on first startup.
